## The Priority Fix — Unify UI + Response Language

Right now there are **two separate language switches:**

```
Switch 1 — App language (Customize panel)
  Controls: button labels, menus, sidebar strings
  Options: English / Hindi only currently

Switch 2 — Globe (response language)
  Controls: what language ARYA responds in
  Options: all 11 Indian languages
```

These should be **one switch.**

```
SINGLE LANGUAGE SELECTOR
  Controls everything simultaneously:
  → UI strings (all 38 keys)
  → ARYA response language
  → Voice output language
  → System prompt language instruction
  → Font for the script
  → RTL direction if needed

User selects Malayalam once
Everything becomes Malayalam
```

**The fix in code:**

```typescript
// language-context.tsx

async function setLanguage(langCode: string) {

  // 1. Update UI strings
  await i18n.changeLanguage(langCode);

  // 2. Update globe / response language
  setSelectedLanguage(langCode);

  // 3. Update document direction
  document.documentElement.dir =
    ['ar', 'he'].includes(langCode) ? 'rtl' : 'ltr';

  // 4. Load correct font
  await loadLanguageFont(langCode);

  // 5. Save to localStorage
  localStorage.setItem('arya_ui_language', langCode);

  // 6. Save to DB
  await fetch('/api/user/preferences', {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      ui_language: langCode,
      response_language: langCode
    })
  });
}

// Banner acceptance
function acceptBanner(detectedLang: string) {
  setLanguage(detectedLang); // one call — changes everything
  dismissBanner();
}

// Globe selection
function onGlobeSelect(langCode: string) {
  setLanguage(langCode); // same one call
}

// Customize panel
function onCustomizeSelect(langCode: string) {
  setLanguage(langCode); // same one call
}
```
