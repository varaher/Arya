## The Root Cause Of All Mobile/Generated Content Issues

**One sentence:** Every server-side content generation function — `morning-briefing.ts`, `weekly-review.ts`, `kaal-briefing.ts`, `patterns-engine.ts` — doesn't pass the user's `ui_language` to the generation prompt.

**The single fix that solves everything:**

```typescript
// server/arya/language-aware-generator.ts

// Add this helper to every generation function:

function getLanguageInstruction(langCode: string): string {
  const instructions: Record<string, string> = {
    hi: `Generate ALL content in Hindi. 
         Warm friend tone. "तुम" not "आप". 
         Natural Hinglish ok for technical terms.
         Keep proper nouns (ARYA, Niti, KAAL, 
         Saturn, Jupiter) in English.`,
    ta: `Generate in Tamil. Warm friend tone.
         Natural conversational Tamil.`,
    ml: `Generate in Malayalam. Warm friend tone.`,
    // ... other languages
    en: `Generate in English.`
  };
  return instructions[langCode] || instructions['en'];
}

// Then in EVERY generation function, add:
const langInstruction = getLanguageInstruction(
  user.ui_language || 'en'
);
// Inject into the prompt before calling Claude
```
