## Issue 1 — Personal Quote Behaviour

### What You Want Back
```
Before:    Quote appears on screen
           Stays visible all day
           Persistent — always there

Now:       Quote in a toggle/accordion
           Disappears after some time
           Has to be tapped to see
```

### The Fix

Find the quote component — likely in the welcome screen or morning briefing section. Look for whatever changed it to a toggle.

```tsx
// WRONG — current version (toggle/collapsible)
const [quoteVisible, setQuoteVisible] = useState(false);

<button onClick={() => setQuoteVisible(!quoteVisible)}>
  Today's thought
</button>
{quoteVisible && (
  <div className="quote">{dailyQuote}</div>
)}

// Or with a timer that hides it:
useEffect(() => {
  setTimeout(() => setQuoteVisible(false), 5000); // ← wrong
}, []);
```

```tsx
// RIGHT — restore to always visible, all day

// Quote should:
// 1. Be visible immediately on load
// 2. Never hide automatically
// 3. Persist all day (same quote, not regenerated)
// 4. Only change the next calendar day

const [dailyQuote, setDailyQuote] = useState('');

useEffect(() => {
  // Load once per day — cached by date
  const today = new Date().toDateString();
  const cached = localStorage.getItem('arya_daily_quote');
  const cachedDate = localStorage.getItem('arya_daily_quote_date');

  if (cached && cachedDate === today) {
    setDailyQuote(cached); // same quote all day
  } else {
    fetchDailyQuote().then(quote => {
      setDailyQuote(quote);
      localStorage.setItem('arya_daily_quote', quote);
      localStorage.setItem('arya_daily_quote_date', today);
    });
  }
}, []);

// Always rendered — no toggle, no timer
return (
  <div className="daily-quote-card">
    <div className="dq-icon">✦</div>
    <div className="dq-text">{dailyQuote}</div>
  </div>
);
```

Tell your developer: **remove the toggle, remove any setTimeout or auto-hide, make the quote a static always-visible card that only refreshes at midnight.**

## Issue 2 — Translation Tone Across All Sections

This is the harder, more important problem.

### Why Maintaining ARYA's Tone In Translation Is Tricky

When Sarvam translates ARYA's English response into Malayalam or Tamil — it translates the words. But it doesn't know:

```
- That ARYA uses tu not vous in French
- That ARYA never uses formal register
- That ARYA speaks like a friend over tea
- That KAAL insights should feel mystical not clinical
- That Niti should feel wise not textbook
- That a mood check-in should feel warm not medical
```

Sarvam's translation engine is accurate — but neutral. It translates what ARYA said, not how ARYA would have said it if it were natively thinking in that language.

---

### How This Plugs Into The System Prompt

```typescript
// server/arya/chat-engine.ts

function buildSystemPrompt(
  user: User,
  detectedLang: string,
  section: string = 'chat'
): string {

  const baseTone = `
You are ARYA — a warm, wise personal thinking companion.
Speak like a knowledgeable friend over tea.
Never like a professor, doctor, or textbook.
Short sentences. Natural phrasing. Human warmth.
  `;

  const languageTone = LANGUAGE_TONE[detectedLang] || '';

  // Section-specific tone for this part of the app
  const sectionTone = getSectionToneInstruction(section, detectedLang);

  return `
${baseTone}

LANGUAGE: ${languageTone}

THIS SECTION (${section}): ${sectionTone}

[rest of prompt — user memory, context, knowledge]
  `.trim();
}
```

### How Sarvam Translation Preserves Tone

When Sarvam translates Claude's English response, it needs the tone context too.

```typescript
// server/arya/sarvam-client.ts

export async function sarvamTranslateWithTone(params: {
  text: string;
  source_language: string;
  target_language: string;
  section: string;
}): Promise<string> {

  // Sarvam's translate API supports a speaker_context field
  // Use it to pass tone instruction

  const toneHint = getSectionToneInstruction(
    params.section,
    params.target_language
  );

  const response = await fetch('https://api.sarvam.ai/translate', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'api-subscription-key': process.env.SARVAM_API_KEY!
    },
    body: JSON.stringify({
      input: params.text,
      source_language_code: toLongCode(params.source_language),
      target_language_code: toLongCode(params.target_language),
      speaker_gender: 'Female',
      mode: 'code-mixed',    // handles natural conversational mixing
      model: 'mayura:v1',
      enable_preprocessing: true,
      // Tone context passed as instruction
      output_script: 'fully-native',
    })
  });

  const data = await response.json();
  return data.translated_text || params.text;
}
```
