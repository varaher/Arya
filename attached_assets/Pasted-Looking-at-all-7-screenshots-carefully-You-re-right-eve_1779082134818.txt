## Why This Is Happening

The main chat screen was translated because it was built with `t()` from the start. These subsections were built later or separately and use hardcoded strings directly.

The fix is mechanical — not architectural. Same pattern for every component:

```tsx
// Before (every subsection currently):
<h1>Sunday Review</h1>
<p>No mood check-ins this week.</p>
<button>Share</button>

// After (what it needs to be):
<h1>{t('review.title')}</h1>
<p>{t('review.no_mood')}</p>
<button>{t('review.share')}</button>
```

Once Hindi is perfect across all 7 screens — the same `t()` wiring works for all 25 languages automatically. Just add the translation file for each language.
